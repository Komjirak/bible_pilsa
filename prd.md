## 1. 서비스 개요

### 한 줄 설명

매일 아침 성경 말씀 한 절을 직접 타이핑하며 믿음을 쌓고, 7일 완주 시 토스 포인트를 받는 기독교인 전용 필사 챌린지 앱.

### 핵심 가치 제안

- **습관 형성**: 큐티(경건의 시간) 루틴에 자연스럽게 연결되는 일일 필사 의식(ritual)
- **보상 설계**: 종교적 성취감(영적 보상) + 토스 포인트 10원(물질적 보상)의 이중 동기 부여
- **진입 장벽 최소화**: 하루 1절, 1~3분 내 완료 가능한 경량 액션
- **저작권 클리어**: 개역한글판(1961) 퍼블릭 도메인 활용 — 법적 리스크 없음

### 서비스명 규격

| 항목 | 값 |
|------|-----|
| 서비스명 (한국어) | 말씀필사 |
| 서비스명 (영문) | BiblePilsa |
| appName (앱인토스 스킴용) | `bible-pilsa` |
| 딥링크 스킴 | `intoss://bible-pilsa` |
| webViewProps.type | `partner` |

## 2. 시장 조사

### 시장 규모 (TAM / SAM / SOM)

| 구분 | 범위 | 규모 | 근거 |
|------|------|------|------|
| TAM | 국내 기독교인 전체 | 약 900만 명 | 통계청 2015 종교 인구 추정, 이후 감소 보정 포함 |
| SAM | 40~60대 기독교인 (스마트폰 사용) | 약 360만 명 | 전체 기독교인의 약 40% 추정, 스마트폰 보급률 95%+ |
| SOM | 토스 앱 사용 40~60대 기독교인 (Year 1 목표) | 약 10만 명 | 토스 누적 사용자 3,000만 × 기독교인 비율 30% × 40~60대 비율 25% × 타겟 전환율 5% |

> 수치는 추정치. 통계청·갤럽 조사 기반 정성적 판단 포함.

### 성장 트렌드

- **디지털 큐티(경건의 시간) 확산**: 코로나 이후 교회 현장 예배 축소 → 개인 경건 생활 디지털화 가속
- **중장년 모바일 퍼스트 전환**: 40~60대의 스마트폰 사용 시간 꾸준한 증가 (2023~2025 방통위 통계 기준 추정)
- **소액 리워드 앱의 성장**: 토스, 리워드앱 시장에서 일상 행동 + 소액 적립 모델 검증 완료
- **성경 앱 꾸준한 성장세**: 갓피플, YouVersion 등 성경 앱 MAU 지속 성장 중 (정성적 관찰)

### 사용자 Pain Point

| Pain Point | 현황 | 우리 서비스의 해결 방식 |
|-----------|------|----------------------|
| 큐티 습관 유지 어려움 | 매일 경건 시간 갖기를 원하지만 귀찮거나 잊어버림 | 토스 앱 안에서 하루 1절, 1~3분 내 완료 가능한 경량 액션 |
| 기존 성경 앱은 '읽기'에 그침 | 수동적 읽기 → 습관화·내면화 어려움 | 직접 타이핑하는 능동적 필사 → 기억에 오래 남음 |
| 필사 보상 없음 | 정신적 성취감 외 즉각 보상 없어 이탈률 높음 | 주 7일 완주 → 토스 포인트 10원 지급 (소액이지만 즉각적·유형적) |
| 디지털 성경 저작권 혼란 | 개역개정판 등 유료 번역본 → 앱 내 사용 법적 리스크 | 개역한글판(퍼블릭 도메인) 채택 → 저작권 이슈 없음 |

### 기회 요인

- 토스 플랫폼의 40~60대 사용자 기반 — 해당 연령대 기독교인 다수 포함
- 앱인토스 포인트 지급 API — "종교 + 소액 리워드" 조합이 국내 미존재
- 개역한글판 퍼블릭 도메인 — 라이선스 비용 없이 1,189장 전체 사용 가능
- 소규모 교회 커뮤니티의 입소문 확산력 — SNS 없이도 오프라인 바이럴 가능

### 위협 요인

- 낮은 ARPU: 포인트 10원/주 → 고수익화 한계, 광고 수익이 핵심 BM
- 경쟁사 업그레이드 가능성: 갓피플 등 기존 성경 앱이 포인트 보상 기능 추가 시 차별화 약화
- 사업자 등록 전제: 포인트·광고 수익화는 사업자 필수 → 비사업자는 즉시 수익화 불가
- 개역한글판 고어체: 일부 젊은 기독교인에게 어색할 수 있으나, 40~60대 타겟에게는 오히려 장점

| **토스 광고 보상 앱들** | 토스 앱인토스 내 | 광고 시청 → 소액 포인트 | 동일 생태계 내 간접 경쟁 — 필사 행동 가치를 더 높게 포지셔닝 |

### 차별화 포인트 (3가지 핵심)

1. **"필사 + 토스 포인트" 결합이 국내 유일**: 성경 필사 앱에 즉각적 금전 보상을 결합한 서비스가 현재 국내 존재하지 않음
2. **토스 생태계 내 마찰 최소화**: 별도 앱 설치 없이 토스에서 바로 실행 → 40~60대 사용자의 앱 설치 거부감 없음
3. **경량 액션 설계**: 하루 1절(30~100자 내외) 필사 → 1~3분 완료 → 죄책감 없는 일일 루프 (기존 앱은 장문 읽기/묵상 강요)

## 4. 전략 수립 및 타겟 설정

### 포지셔닝

> "토스에서 하루 한 절, 말씀을 손으로 쓰고 포인트를 받는 기독교인의 아침 습관 앱"

### 핵심 타겟 세그먼트

| 구분 | 설명 |
|------|------|
| **1차 타겟 (Primary)** | 40~60대 기독교인 토스 사용자 — 매일 큐티 습관 있거나 원하는 사람, 소액 포인트 적립에 친숙한 사람 |
| **2차 타겟 (Secondary)** | 30대 직장인 기독교인 — 교회 출석하지만 개인 경건 시간이 부족한 사람, 앱인토스 헤비 유저 |

### Go-to-Market 전략

| 채널 | 전략 |
|------|------|
| 앱인토스 스토어 자연 유입 | 부제·설명·검색 키워드 최적화, 완주 스크린샷 우선 노출 신청 |
| 교회 커뮤니티 바이럴 | 필사 완료 후 카카오톡·밴드 공유 버튼 → 교회 단톡방 확산 유도 |
| 토스 홈 광고 | 상세 설명 충실 기입 → 토스 홈 광고 선정 대상 우선순위 확보 |
| 초기 리텐션 강화 | 첫 7일 달성 시 첫 포인트 지급 경험 → 다음 주 재참여 동기 고착화 |

### 성장 지표 (KPI)

| 지표 | 목표 (3개월) | 목표 (12개월) |
|------|-------------|--------------|
| MAU | 10,000명 | 100,000명 |
| DAU/MAU | 35% 이상 | 40% 이상 |
| 주간 완주율 (7일 완주/월간 활성) | 25% 이상 | 30% 이상 |
| 광고 수익 (월) | 150만 원 이상 | 1,500만 원 이상 |
| 앱인토스 스토어 평점 | 4.2 이상 | 4.5 이상 |

### MVP 전략

- **MVP 범위**: 오늘의 말씀 랜덤 1절 + 필사 입력 + 유사도 검사(85%) + 주간 달력 + 포인트 지급
- **비포함 (v2)**: 소셜 공유, 구절 즐겨찾기, 읽기 기록 아카이브, 번역본 선택
- **검증 기준**: 3개월 DAU/MAU 35% 달성 → 사용자 습관 형성 확인 후 v2 기능 추가

## 5. 타겟 사용자 (페르소나)

### 페르소나 A — 이정숙 집사 (55세, 주부)

- **배경**: 30년 신앙생활, 매주 교회 출석, 성경 필사 노트 1권 보유 중
- **상황**: 아침 6시 30분 기도와 성경 읽기 후 토스로 가계부 확인하는 루틴
- **Pain Point**: "성경 필사 하고 싶은데 손이 아파서 오래 못 하겠어", "앱 새로 받는 게 귀찮아"
- **동기**: 작은 포인트라도 쌓이면 뿌듯함, 교회 친구들에게 자랑하고 싶음
- **사용 시나리오**: 아침 기도 후 토스 열기 → 말씀필사 실행 → 오늘 말씀 1절 필사 → 완료 → 광고 시청 (보상형) → 포인트 확인 → 카카오톡 공유

### 페르소나 B — 김민준 집사 (42세, 직장인)

- **배경**: 교회 청장년부 소속, 주일 예배는 가지만 평일 큐티는 잘 못 함
- **상황**: 출근 지하철에서 토스 확인하는 습관 있음
- **Pain Point**: "큐티 해야 하는데 성경 찾기도 귀찮고, 앱 따로 안 받고 싶음"
- **동기**: 7일 완주 포인트 + 주간 스트릭 달성 시 성취감
- **사용 시나리오**: 출근 지하철에서 토스 알림 확인 → 말씀필사 진입 → 1분 내 필사 완료 → 이번 주 6/7일 달성 확인 → 앱 종료

## 6. 핵심 기능 목록

### Must (MVP 필수)

| 기능 | 설명 |
|------|------|
| M-01. 오늘의 말씀 랜덤 제공 | 매일 개역한글판 성경 1절 랜덤 선정 (서버 기준 날짜, 동일 날짜 동일 구절) |
| M-02. 오버레이 필사 UI | **원문 텍스트를 배경에 연한 회색으로 표시하고, 사용자가 타이핑하면 글자 위에 덮어쓰는 방식**. 별도 입력 필드 없음. 올바른 글자는 검정색으로 덮어씌워지고, 오타는 빨간색으로 즉시 표시. 오프라인 필사장 경험 재현. |
| M-03. 실시간 글자 검증 | 글자 단위 실시간 비교. 정확한 글자 → 검정/진하게 덮어쓰기. 오타 → 빨간 하이라이트. 전체 완료 후 최종 유사도 85% 이상 시 통과. |
| M-04. 주간 달력 뷰 | 이번 주 7일 완료/미완료 시각화 (월~일 기준) |
| M-05. 주간 완주 포인트 지급 | 7일 누적 완료 시 토스 포인트 10원 지급 (BE Point API 연동) |
| M-06. 광고 노출 | 필사 완료 직후 전면 광고 1회 노출 (IAA) |
| M-07. 토스 로그인 연동 | appLogin() → accessToken → userKey 기반 사용자 식별 |
| M-08. 출처 표기 | 화면 하단 "성경 번역 출처: 대한성서공회 개역한글판" 상시 노출 |
| M-09. 개인정보처리방침 | 앱 설정 > 개인정보처리방침 접근 가능 |

## 7. 사용자 시나리오

### 주요 시나리오 — 일일 필사 완료 플로우

```
[진입]
토스 앱 실행 → 앱인토스 탭 → "말씀필사" 선택
또는 푸시 알림 탭 → 앱 직접 진입

[인증]
첫 실행 시: 토스 로그인 (appLogin()) → 약관 동의 → 완료
재방문 시: 자동 로그인 (refreshToken)

[메인 화면]
오늘의 말씀 표시 (성경 구절 + 책/장/절 정보)
이번 주 달력 (완료일 체크 표시)
"지금 필사하기" 버튼

[필사 화면 — 오버레이 방식]
화면 전체: 원문 말씀을 연한 회색(투명도 낮은)으로 대형 폰트 표시
사용자 타이핑: 원문 글자 위에 글자를 하나씩 덮어씀
  - 정확한 글자 → 검정색(진하게)으로 덮어씌워져 원문 사라지는 효과
  - 오타 → 해당 글자 빨간색 하이라이트 즉시 표시
별도 입력 필드 없음 — 화면 탭 시 키보드 올라오는 방식
복사/붙여넣기 비활성화
진행률: 상단 프로그레스 바 (0~100%)

[검증]
"완료" 클릭 → 유사도 검사 (서버 또는 클라이언트)
통과 (85% 이상): 오늘 말씀 완료 처리 → 달력 체크 ✅
미통과 (85% 미만): "조금 더 정확하게 입력해주세요" 안내 → 재입력 기회

[광고 노출]
완료 처리 후 → 전면 광고 자동 노출 (Skip 5초 후 가능)

[완료 화면]
오늘 완료! 메시지 + 주간 진행 현황 (예: "이번 주 5/7일 완료")
7일 완주 시: "🎉 주간 완주! 토스 포인트 10원 적립" 팝업
포인트 지급 API 호출 → 완료 확인
```

### 예외 시나리오 1 — 오늘 이미 완료한 경우

## 8. 앱인토스 정책 준수 체크리스트

앱인토스 가이드(`apps-in-toss-guide.md`) 기준.

| 항목 | 판정 | 근거 |
|------|------|------|
| 금지 서비스 해당 여부 (가상자산, 사행성, 금융중개 등) | PASS | 종교 콘텐츠 앱 — 금지 목록 해당 없음 |
| 비사업자 수익화 제한 준수 | PASS (조건부) | 사업자 등록 전제 하에 포인트·IAA 사용 가능 |
| 토스 로그인 연동 (사업자 필수) | PASS (조건부) | 사업자 등록 후 OAuth2 연동 |
| TDS (Toss Design System) 적용 | PASS | 비게임 WebView 미니앱 → TDS 필수 적용 예정 |
| iframe 사용 금지 | PASS | iframe 사용 계획 없음 |
| 앱 로고 규격 600×600px PNG (배경색 필수) | PASS | 디자인 에이전트가 규격에 맞게 제작 예정 |
| 정방형 썸네일 1000×1000px | PASS | 디자인 에이전트 제작 예정 |
| 가로형 썸네일 1932×828px | PASS | 디자인 에이전트 제작 예정 |
| appName 규칙 (15자 이내, 명사형) | PASS | `bible-pilsa` — 규칙 준수 |
| 앱 번들 100MB 이하 | PASS | 성경 텍스트 ~4MB + UI 번들 → 100MB 여유 |
| SDK 2.0.1 (`@apps-in-toss/web-framework@2.0.1`) | PASS | 최신 SDK 기반 개발 예정 |
| 빌드 커맨드 `ait build` 사용 | PASS | SDK 2.0.1 기준 적용 |
| 만 19세 이상 이용 제한 준수 | PASS | 앱인토스 플랫폼 자체 연령 제한 |
| 개인정보처리방침 등록 | PASS (조건부) | 출시 전 필수 준비 — 필사 이력 저장에 따른 법적 의무 |
| 저작권 문제 없는 콘텐츠 | PASS | 개역한글판 퍼블릭 도메인 확인됨, 출처 표기 "대한성서공회" 필수 |
| CORS 허용 설정 | PASS | `{appName}.apps.tossmini.com` Origin 허용 예정 |
| 복권/베팅성 요소 없음 | PASS | 랜덤 말씀 제공이나 포인트 지급은 베팅 요소 없음 |

## 9. 비기능 요구사항

### 성능

| 항목 | 목표 |
|------|------|
| 앱 초기 로딩 | 2초 이내 (TDS 컴포넌트 지연 없는 핵심 화면) |
| 필사 유사도 검사 응답 | 1초 이내 (클라이언트 사이드 처리 우선) |
| 포인트 지급 API 응답 | 5초 이내 (실패 시 재시도 큐) |
| 앱 번들 크기 | 압축 해제 100MB 이하 (성경 텍스트 ~4MB, 나머지 UI) |

### 보안

| 항목 | 요구사항 |
|------|---------|
| 인증 | 토스 OAuth2 기반 (appLogin, refreshToken) |
| 통신 암호화 | HTTPS + mTLS (앱인토스 API 통신) |
| Fraud 방지 | 타이핑 속도 제한, 복사/붙여넣기 비활성화, 서버 중복 완주 방지 |
| 개인정보 | 최소 수집 원칙 (userKey, 필사 완료 날짜, 포인트 이력만 저장) |

### 접근성

- 폰트 크기: 최소 16px (40~60대 타겟 고려, 본문 18~20px 권장)
- 버튼 터치 영역: 최소 44×44px
- 색상 대비: WCAG AA 수준 (4.5:1 이상)
- 개역한글판 고어체 이해 보조: 어려운 단어 롱프레스 시 현대어 풀이 (v1.1 이후 검토)

### 호환성

| 플랫폼 | 최소 버전 |
|--------|---------|
| Android OS | Android 7 (앱인토스 기준) |
| iOS | iOS 16 (앱인토스 기준) |
| 개발 방식 | WebView (`@apps-in-toss/web-framework@2.0.1`) |

### 다크 모드

- 토스 앱 다크 모드 설정에 연동하여 자동 전환
- TDS 색상 토큰 기반으로 다크 모드 대응 (`bridgeColorMode: 'basic'`)
- 성경 원문 텍스트 색상: 라이트 `#3182F6` / 다크 `#5A9FFF` (TDS 블루 계열)

## 10. 제외 범위 (Out of Scope)

다음 항목들은 MVP 범위에 포함하지 않는다:

| 항목 | 이유 |
|------|------|
| 다국어 지원 (영문 등) | 국내 40~60대 한국어 전용 타겟, 해외 확장은 v3 이후 검토 |
| 음성 필사 (STT) | 핵심 가치가 "타이핑"을 통한 능동적 필사임 — 음성은 의도와 맞지 않음 |
| 손글씨 인식 (OCR) | 기술 복잡도 과다, MVP 범위 초과 |
| 사용자 간 커뮤니티 기능 | 소셜 플랫폼 규제(소셜·만남 플랫폼 금지) 가능성 → v2 이후 신중 검토 |
| 개역개정판 등 유료 번역본 | 라이선스 협의 필요 → 성장 단계 이후 추가 |
| 인앱 결제 (IAP) | 현 BM에서 불필요, 추후 프리미엄 버전 시 검토 |
| 필사 내용 노트 기능 | 핵심 플로우 집중을 위해 제외, v2 이후 검토 |
| 관리자 대시보드 | 운영 초기에는 앱인토스 콘솔 + 서버 로그로 대체 |
