# Changelog

## [1.1.0] - 2026-03-10

### ✨ Features & Business Model
- **오늘의 보상형 전면 광고 연동:** 
  - 필사 완료 화면의 "완료" 버튼을 "광고 시청 후 포인트 받기"로 고도화.
  - 포인트 수령 전 `ait.v2.live.6255b174cfea47ca` 전면 광고 시청 흐름 강제화로 비즈니스 모델 강화.
- **성경 순차 필사 모드 추가:**
  - "오늘의 말씀(랜덤)" 외에 창세기부터 순차적으로 필사할 수 있는 "순서대로" 모드 신규 런타임 구현.
  - `SequentialProgress` 컴포넌트 및 로컬/서버 연동 진도 관리 로직 추가.

### 🎨 UI/UX Improvements
- **설정 저장 피드백 (Toast):**
  - 글자 크기, 알림 설정 변경 시 즉각적인 "설정이 저장되었습니다" 토스트 알림 추가.
  - 설정 페이지 내 푸시 알림 수신에 대한 정책 가이드 (광고성 정보 안내) 문구 보강.
- **필사 오타 시각화 고도화:**
  - 오타 발생 시 단순히 빨간색 표시가 아닌, **실제로 입력한 오타 글자**를 빨간색으로 오버레이하여 직관적인 교정 피드백 제공.
- **설정 동기화 강화:**
  - 기존 로컬 전용이었던 `fontSize` 설정을 백엔드 DB(`User` 모델)와 동기화하여 기기 변경 시에도 유지되도록 개선.

### 🛠 Bug Fixes & Optimization
- **배너 광고 로딩 안정화:**
  - `window.TossAds` 객체 접근 방식을 개선하여 일부 환경에서 배너가 노출되지 않던 현상 해결.
- **필사 시간 제한(Timeout) 완화:**
  - 긴 구절 필사 시 "완료 처리 중 오류"가 발생하던 서버측 `durationMs` 검증 로직을 60초에서 10분으로 확장.
- **백엔드 빌드 안정화:**
  - `toss-crypto.util.ts` 등에서 발생하던 TypeScript 타입 추론 오류 및 `null` 반환 이슈 수정.
- **프로젝트 정리:**
  - 불필요한 설계 문서(prd, prange 등) 및 중복 에셋 파일 일괄 제거로 코드베이스 경량화.

## [1.0.0] - 말씀필사 (Bible-Pilsa) Apps-in-Toss Release

### ✨ Features
- **Toss 로그인 연동 내재화:**
  - AES-256-GCM 알고리즘을 사용한 사용자 E-mail 복호화 로직 백엔드 유틸리티(`toss-crypto.util.ts`) 추가 처리 완료.
- **7일 완주 리워드 연동 (executePromotion):**
  - 기존 mTLS 기업 인증망 통신에서 권장 규격인 `executePromotion` (Bearer + `x-toss-user-key`) 방식으로 `PointService` 전면 개편.
  - 7일 차 말씀 필사 완료 시 자동으로 토스 10원 리워드 지급 API 발송 로직 체결.
- **Toss Ads 인앱광고 2.0 연동:**
  - **홈 화면:** 화면 하단 고정 영역에 `AdBanner` 컴포넌트 추가 (`ait.v2.live.ae8e04b2200544f5`).
  - **완료 화면:** 오늘의 말씀을 성공 제출 후 `홈으로` 복귀 시 전면 인터스티셜(Interstitial) 광고 송출 후 전환 동작 연계 (`ait.v2.live.6255b174cfea47ca`).
- **서비스 약관 및 설정 화면:**
  - `Terms of Service`(서비스 이용약관) HTML 페이지 자체 레포지토리 배포.
  - 설정 화면 내 법적 고지 탭 이름 변경 및 라우팅 추가, 성경 소스본 출처 (개역한글 1961) 명시.

### 🎨 UI/UX & Design System
- **TDS (Toss Design System) 마이그레이션:**
  - 기존 낡은 주황색 원고지(Warm Paper) 테마에서 탈피하여, 화이트(밝은 톤) 배경과 토스 블루(`#3182F6`) 포인트 컬러의 정식 TDS 레이아웃으로 전면 개편.
- **말씀 카드 UI 고도화:**
  - `TodayVerseCard`의 레이아웃을 실제 성경책 도서 모형(책등 두께, 양피지 제본선, 리본 책갈피) 구조로 변경하여 퀄리티 향상.
- **App-in-Toss 글로벌 네비게이션 준수:**
  - 토스 앱 네이티브 헤더와의 충돌을 막기 위해 자체 웹 DOM 기반 `<` 형태의 물리 뒤로가기 버튼을 일괄 제거 (`AppNavBar`).

### 📦 Chore
- `@apps-in-toss/web-framework` SDK 버전 `v2.0.1` 호환성 업데이트 완료.
